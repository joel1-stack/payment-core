⚡ Payments Core

An immutable, double-entry ledger for multi-party payment splits.

❌ The Problem

Most marketplaces and SaaS platforms split payments using simple database updates. If a network glitch causes a payment gateway (Stripe, M-Pesa) to send the same webhook twice, the merchant gets double-credited. If a refund happens, the ledger breaks.

✅ The Solution

This engine treats every payment like a bank. It uses Double-Entry Bookkeeping. Every cent moved is recorded as a DEBIT in one account and a CREDIT in another. If Debits don't exactly equal Credits, the transaction is rejected.
